DMAP abra 플러그인의 컴포넌트 구성도를 보여주는 기술 다이어그램.

"abra 플러그인 — Dify DSL 자동화" 타이틀

디렉토리 트리 시각화 (좌측):
abra/
├── .claude-plugin/ (plugin.json, marketplace.json)
├── skills/ (8개 스킬)
├── agents/ (5개 에이전트)
│   ├── scenario-analyst/
│   ├── dsl-architect/
│   ├── plan-writer/
│   ├── prototype-runner/
│   └── agent-developer/
├── gateway/ (install.yaml, runtime-mapping.yaml)
├── commands/ (슬래시 명령)
└── README.md

컴포넌트 관계도 (우측):
- Skills 영역 (연보라색):
  - setup, core (필수/기본)
  - dify-setup, scenario, dsl-generate, prototype, dev-plan, develop (기능 스킬)

- Agents 영역 (연파란색):
  - scenario-analyst: 시나리오 분석 전문가
  - dsl-architect: DSL 설계 전문가
  - plan-writer: 개발계획 작성 전문가
  - prototype-runner: 프로토타입 실행 전문가
  - agent-developer: Agent 개발 전문가

- Gateway 영역 (연초록색):
  - Dify CLI, DSL 검증 도구 등 커스텀 도구 연결

사용자 플로우 화살표:
"시나리오 생성" → "DSL 생성" → "프로토타이핑" → "개발계획" → "Agent 개발"

스타일: 깔끔한 컴포넌트 다이어그램, 둥근 모서리, 파스텔 색상, 한글 텍스트. 흰색 배경.
