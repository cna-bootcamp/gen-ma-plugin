DMAP(선언형 멀티에이전트 플러그인) 아키텍처 다이어그램.

Clean Architecture 패턴 기반의 계층 구조를 보여주는 깔끔한 기술 다이어그램.

구성 요소 (위에서 아래로 흐르는 레이어 구조):
1. 최상단: "사용자 요청" (Input) — 파란색 화살표로 진입
2. 두번째 층: "Skills (Controller)" — 라우팅 + 오케스트레이션, 연보라색 박스
   - 위임형 스킬: Core, Planning, Orchestrator
   - 직결형 스킬: Setup, Utility
3. 세번째 층: "Agents (Service)" — 전문가 자율 실행, 연파란색 박스
   - architect, executor, explorer 등 역할별 아이콘
4. 네번째 층: "Gateway" — 추상→구체 매핑, 연초록색 박스
   - runtime-mapping.yaml 표시
5. 최하단: "Runtime" — 실행 환경, 연회색 박스
   - Claude Code, Codex CLI 등 로고
6. 오른쪽 옆에: "리소스 마켓플레이스" — 공유 자원 풀, 연노란색 별도 박스
   - 가이드, 템플릿, 샘플, 도구

화살표: Skills → Agents → Gateway → Runtime (단방향 의존)
직결형 스킬은 점선으로 Agents를 건너뛰고 Gateway에 직접 연결

스타일: 깔끔한 플랫 디자인, 둥근 모서리 박스, 얇은 연결선, 밝은 파스텔 색상.
