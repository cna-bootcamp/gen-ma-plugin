DMAP 프롬프트 조립(Prompt Assembly) 과정을 보여주는 기술 다이어그램.

왼쪽에서 오른쪽으로 흐르는 3단계 조립 과정:

1단계 "에이전트 패키지 로드":
- agents/architect/ 디렉토리에서 3개 파일 아이콘
  - AGENT.md (마크다운 아이콘, 파란색) — "목표 + 워크플로우"
  - agentcard.yaml (YAML 아이콘, 초록색) — "역할 + 제약"
  - tools.yaml (YAML 아이콘, 노란색) — "필요 도구"

2단계 "Gateway 참조로 구체화":
- runtime-mapping.yaml을 참조하는 화살표
  - tier: HIGH → claude-opus-4-6 (모델 구체화)
  - file_read → Read, Glob (도구 구체화)
  - forbidden: file_write → Write, Edit 제외 (금지 구체화)

3단계 "최종 프롬프트 조립":
- 하나로 합쳐진 큰 문서 아이콘
  - 공통 정적 (runtime-mapping) 영역
  - 에이전트별 정적 (3파일 합침) 영역
  - 동적 (작업 지시) 영역
- "Task(FQN, model, prompt) 호출" 텍스트

스타일: 왼→오 흐름 다이어그램, 깔끔한 플랫 디자인, 파스텔 색상, 한글 텍스트. 흰색 배경.
