## 개요

DMAP 생태계는 **DMAP 빌더 → AI팀(플러그인) → 외부 플러그인 → AI 서비스**의 계층으로 구성됨.
DMAP 빌더가 목적별 AI팀(플러그인)을 생성하고,
각 AI팀은 **필요한 외부 플러그인을 자유롭게 탈착하여 역량을 확장**하는 구조.

스마트폰에 앱을 설치/삭제하듯,
AI팀(플러그인)에 외부 역량을 탈착하는 방식으로 동작함.

## 생태계 역할 정의

| 현실 세계 비유 | DMAP 생태계 | 역할 |
|--------------|------------|------|
| 인력파견 회사 | DMAP 빌더 | 멀티에이전트 플러그인 생성기 |
| 프로젝트 투입 팀 | 목적별 플러그인 | AI팀 (도메인 전문가 집단) |
| 팀이 사용하는 외부 도구 | 외부 플러그인 (ext-{}) | 필요에 따라 탈착하는 확장 모듈 |
| 팀이 납품한 서비스 | AI 서비스 | AI팀이 만든 결과물 |

### 핵심 원칙

- **AI팀은 도메인 지식에 집중** — 공통 기능은 외부 플러그인에 위임
- **외부 플러그인은 자유롭게 탈착** — add/remove-ext-skill로 필요할 때 연결, 불필요 시 제거
- **선언형 A2A로 연동** — 인프라 없이 마크다운 기반 계약으로 협업
- **생태계는 성장함** — 새로운 목적별 플러그인이 지속적으로 추가 가능

## 구조도

```
┌─────────────────────────────────────────────┐
│              DMAP 빌더                       │
│         (멀티에이전트 플러그인 생성기)          │
└──────────────────┬──────────────────────────┘
                   │ 생성
        ┌──────────┼──────────┐
        ▼          ▼          ▼
  ┌──────────┐ ┌──────────┐ ┌──────────┐
  │blog-poster│ │curriculum│ │   ...    │
  │  AI팀    │ │  AI팀    │ │  AI팀    │
  └────┬─────┘ └────┬─────┘ └────┬─────┘
       │             │            │
       │  ext-{}     │  ext-{}    │  ext-{}
       │  탈착       │  탈착      │  탈착
       ▼             ▼            ▼
  ┌─────────────────────────────────────────┐
  │         외부 플러그인 풀                  │
  │  ┌───────┐ ┌────────────────────┐       │
  │  │ Abra  │ │github-release-mgr  │  ...  │
  │  │(개발) │ │(릴리스)            │       │
  │  └───────┘ └────────────────────┘       │
  └─────────────────────────────────────────┘
```

각 AI팀은 외부 플러그인 풀에서 필요한 모듈만 선택적으로 연결함.
모든 AI팀이 동일한 외부 플러그인을 사용할 필요 없음.

## 외부 플러그인 탈착 (Plug & Unplug)

### 연동 방식

외부 플러그인은 **ext-{} 스킬 패턴**(선언형 A2A)으로 연동됨.
마크다운 기반 명세서가 계약 역할을 하므로 별도 인프라 구축이 불필요함.

| 동작 | 명령 | 설명 |
|------|------|------|
| 추가 | `/add-ext-skill` | 마켓플레이스에서 명세서 다운로드 → ext-{} 스킬 자동 생성 |
| 제거 | `/remove-ext-skill` | ext-{} 스킬 및 관련 명세서 깔끔하게 제거 |

### 현재 등록된 외부 플러그인

| 외부 플러그인 | 역할 | 사용 예시 |
|-------------|------|----------|
| abra | AI 서비스 개발 도구 (Dify/Python) | Dify DSL 생성, 프로토타이핑, Agent 개발 |
| github-release-manager | GitHub Release 관리 | Release 문서 생성, 수정, 삭제 |

### 향후 추가 가능한 외부 플러그인

| 외부 플러그인 | 역할 | 사용 예시 |
|-------------|------|----------|
| code-reviewer | 코드 리뷰 자동화 | PR 리뷰, 코드 품질 분석 |
| test-generator | 테스트 자동 생성 | 단위 테스트, 통합 테스트 생성 |
| deploy-manager | 배포 파이프라인 관리 | CI/CD, 인프라 프로비저닝 |

### 탈착의 의미

AI팀마다 필요한 외부 플러그인 조합이 다름.

```
blog-poster AI팀:     ext-abra (개발) + ext-github-release-manager (릴리스)
curriculum AI팀:      (외부 플러그인 없음 — 자체 스킬만으로 동작)
새로운 AI팀:          ext-abra (개발) + ext-code-reviewer (리뷰) + ...
```

필요할 때 붙이고, 불필요하면 떼는 구조.
이것이 "Plug & Unplug"의 핵심.

## Abra: AI 서비스 개발 도구

Abra는 현재 DMAP 생태계에서 **유일한 AI 서비스 개발 도구** 플러그인.
Dify 워크플로우와 Python 기반의 두 가지 개발 파이프라인을 제공함.

| 유형 | 설명 | 특징 |
|------|------|------|
| Dify Agent AI 서비스 | Dify 워크플로우 기반 AI 서비스 | 노코드, 빠른 프로토타이핑 |
| Python AI 서비스 | Python 코드 기반 AI 서비스 | 유연한 커스터마이징 |

향후 목적별로 더 전문화된 개발 플러그인이 추가될 수 있음.
예를 들어 Node.js 기반 개발 도구, 특정 프레임워크 전용 빌더 등.
Abra는 그때에도 Dify/Python 영역의 핵심 개발 도구로서 공존하는 구조.

## 구체 예시

### 예시 1: blog-poster 플러그인 (외부 플러그인 2개 탈착)

```
[DMAP 빌더] → blog-poster 플러그인 생성

blog-poster AI팀 구성:
  - 콘텐츠 기획자 에이전트
  - 콘텐츠 작성자 에이전트
  - 편집자 에이전트
  - 배포자 에이전트

외부 플러그인 탈착:
  /add-ext-skill abra                    → ext-abra 스킬 생성
  /add-ext-skill github-release-manager  → ext-github-release-manager 스킬 생성

서비스 개발:
  "매일 자동으로 블로그 포스팅하는 서비스 만들어줘"
       ↓ ext-abra 호출
  [Abra 플러그인] → Dify Agent AI 서비스 생성

릴리스 관리:
  "v1.0.0 릴리스 문서 만들어줘"
       ↓ ext-github-release-manager 호출
  [github-release-manager] → Release 문서 생성
```

### 예시 2: gen-ai-curriculum 플러그인 (외부 플러그인 없이 동작)

```
[DMAP 빌더] → gen-ai-curriculum 플러그인 생성

gen-ai-curriculum AI팀 구성:
  - 교재 분석 에이전트
  - 커리큘럼 작성 에이전트
  - 프리젠테이션 작성 에이전트

외부 플러그인: 없음 (자체 스킬만으로 동작)

서비스 실행:
  "교재 업로드하면 커리큘럼 자동 생성해줘"
       ↓ 자체 에이전트 협업
  교재 분석 → 커리큘럼 생성 → 프리젠테이션 작성
```

### 예시 3: api-service 플러그인 (다중 외부 플러그인 조합)

```
[DMAP 빌더] → api-service 플러그인 생성

api-service AI팀 구성:
  - API 설계자 에이전트
  - 개발자 에이전트
  - 테스터 에이전트
  - 릴리스 매니저 에이전트

외부 플러그인 탈착:
  /add-ext-skill abra                    → 서비스 개발용
  /add-ext-skill github-release-manager  → 릴리스 관리용
  /add-ext-skill code-reviewer (향후)     → 코드 리뷰용

각 에이전트가 필요한 시점에 해당 외부 플러그인을 호출하여 협업
```

## 향후 방향

1. **외부 플러그인 생태계 확장** — 목적별 전문 플러그인 지속 추가
   (코드 리뷰, 테스트 생성, 배포 관리 등)
2. **플러그인 조합 패턴** — 자주 사용되는 외부 플러그인 조합을 템플릿화
3. **커뮤니티 기여 플러그인** — 서드파티 개발자가 외부 플러그인을 만들어 마켓플레이스에 등록
4. **다중 런타임 지원** — Claude Code 외 Codex CLI, Gemini CLI 등으로 확장
